# Swag Labs - тестовый магазин
## Архитектура: 

```
Swag_labs/
├── pages/
│   ├── inventoryPage/   
│   │ ├── __init__.py       # Файл с классом для работы с страницей товаров
│   │ └── locators.py       # Файл содержащий локаторы элементов страниц
│   ├── loginPage/       
│   │ ├── __init__.py       # Файл с классом для работы с страницей входа
│   └ └── locators.py       # Файл содержащий локаторы элементов страниц
├── browser.py              # Файл с данными о драйвере/ах
├── conftеst.py             # Файл с конфигурационными данными, например, URL
├── tests.py                # Файл с тестами для автоматизированного тестирования
└── Readme.md               # Файл с описанием проекта и инструкциями
```

## Описание:

1. **`sausedemo/`**: Главная папка проекта.

2. **`pages/`**: Папка, содержащая файлы, связанные с веб-страницами.
    - **`inventoryPage/`**: Папка страницы товаров
    - - **`__init__.py`**: Файл с классом для работы с страницей товаров
    - - **`locators.py`**: Файл, содержащий локаторы элементов страниц. Локаторы используются для поиска и взаимодействия с элементами на веб-страницах.

    - **`loginPage/`**: Папка страницы входа.
    - - **`locators.py`**: Файл, содержащий локаторы элементов страниц. Локаторы используются для поиска и взаимодействия с элементами на веб-страницах.
    - - **`__init__.py`**: Файл с классом (или классами) для работы с страницей входа. Здесь могут быть методы для заполнения формы логина, отправки данных и проверки результатов.
   
3. **`browser.py`**: Файл с данными о драйвере. Здесь может быть код по работе и управлению драйвером/ами и т. д.

4. **`conftest.py`**: Файл с конфигурационными данными. Здесь может быть информация о URL-адресах, учетных данных, настройках тестирования, фикстур и т. д.

5. **`tests.py`**: Файл с автоматизированными тестами. Здесь могут быть реализованы тест-кейсы, использующие классы и методы из файлов в папке `pages/`.

6. **`Readme.md`**: Файл с описанием проекта и инструкциями. Здесь могут быть указаны цели проекта, требования, инструкции по установке и запуску тестов, а также другая полезная информация для разработчиков и пользователей проекта.

Эта структура проекта помогает организовать код и ресурсы таким образом, чтобы проект был легко понятен и масштабируем.
